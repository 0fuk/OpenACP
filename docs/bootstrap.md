# ACCP Bootstrap

ACCP Bootstrap helps teams move from rough intent to a minimal work package.

## Use Bootstrap When

- there is only a rough product idea,
- a PRD has no spec,
- scope boundaries are unclear,
- no task cards exist,
- no verification plan exists,
- implementation would require guessing.

## Outputs

- PRD intake,
- source pack,
- source manifest,
- scope boundary,
- assumptions ledger,
- open questions,
- human decision queue,
- starter specs,
- first task cards.

## Stop Before Implementation When

- the workflow is ambiguous,
- the actor is unknown,
- acceptance cannot be measured,
- real credentials or real user data are needed,
- external side effects are requested,
- no safe default exists.
